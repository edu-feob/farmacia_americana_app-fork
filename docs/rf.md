## Requisitos Funcionais (RF)
O que o sistema deve fazer para ser funcional.


## MVP 1: Jornada de Compra e Catálogo

###  RF01 - Catálogo Dinâmico 
O sistema deve listar produtos com nome, imagem, preço e categoria.

### RF02 - Busca Inteligente
O usuário deve conseguir pesquisar itens por nome ou categoria com correção ortográfica básica.

### RF03 - Carrinho de Compras
O sistema deve permitir adicionar, remover e alterar quantidades de produtos.  

### RF04 - Gestão de Categorias
O app deve segmentar produtos em: MIPs, Suplementos, Higiene e Medicamentos Controlados.


## MVP 2: Atendimento e IA (O Diferencial)

### RF05 - Chatbot de Triagem
O sistema deve ter uma interface de chat para tirar dúvidas sobre medicamentos isentos de prescrição (MIPs).
### RF06 - OCR de Receitas
O usuário deve poder carregar uma foto da receita; o sistema deve extrair o texto e identificar os medicamentos correspondentes no estoque.

### RF07 - Protocolo de Atendimento
Cada interação no chat deve gerar um número de protocolo para continuidade humana, se necessário.


## MVP 3: Checkout e Recorrência

### RF08 - Método de Pagamento 
O sistema deve possuir opções de escolha de pagamentos via PIX e Cartão de Crédito (API de terceiros).

### RF09 - Cadastro
O sistema deve integrar uma função de criação de conta e ou login com conta no Facebook/Google
